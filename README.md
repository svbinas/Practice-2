# Practice-2

**Последовательные и параллельные алгоритмы сортировки с использованием OpenMP**

---

## Описание проекта

Данный репозиторий содержит реализацию и сравнение **последовательных** и **параллельных** алгоритмов сортировки на языке **C++** с использованием технологии **OpenMP**.

Цель задания — изучить особенности классических алгоритмов сортировки, попытаться распараллелить их с помощью OpenMP, измерить производительность и проанализировать влияние параллелизма на время выполнения.

---

### 1. Реализация сортировок без параллелизма

Реализованы следующие алгоритмы сортировки в последовательном варианте:
- сортировка пузырьком (Bubble Sort);
- сортировка выбором (Selection Sort);
- сортировка вставкой (Insertion Sort).

Каждый алгоритм реализован в виде отдельной функции без использования OpenMP.

---

### 2. Параллельная реализация с использованием OpenMP

Для ускорения вычислений были реализованы параллельные версии алгоритмов с использованием директив OpenMP:

- **Пузырьковая сортировка** — реализована в виде odd-even transposition sort, что позволяет корректно распараллелить сравнение независимых пар элементов.
- **Сортировка выбором** — распараллелен этап поиска минимального элемента на каждом шаге.
- **Сортировка вставкой** — реализована параллельная версия через разбиение массива на блоки, локальную сортировку вставкой и последующее параллельное слияние блоков.

Для распараллеливания использовались директивы:
- `#pragma omp parallel for`
- `#pragma omp parallel`
- `#pragma omp critical`

---

### 3. Сравнение производительности

Для сравнения производительности:
- использована библиотека `<chrono>` для измерения времени выполнения;
- протестированы массивы следующих размеров:
  - **1000 элементов**
  - **10 000 элементов**
  - **100 000 элементов**
- измерено время выполнения:
  - последовательных версий сортировок;
  - параллельных версий сортировок.

Также выполнялась проверка корректности сортировки с помощью `std::is_sorted`.
